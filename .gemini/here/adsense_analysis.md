## 1. 근본 원인 분석

### 🚨 원인 ①: 가치 없는 콘텐츠 (Low-Value Content)로 인한 거절
구글 애드센스는 사이트에 **독창적인 텍스트 콘텐츠**와 **실제 사용자 참여**가 충분한지 심사합니다.
* **DB 데이터 분석 결과**:
  * **작성된 게시글 수**: `5,024개`
  * **등록된 전체 댓글 수**: `71개` (게시글 대비 댓글 비율 약 **1.4%**)
  * **가입된 유저 프로필 수**: `39개`
* **진단**: 5,000개가 넘는 많은 게시글이 등록되어 있으나 댓글은 거의 존재하지 않습니다. 이는 크롤러가 보기에는 **자동 RSS 피드 수집 또는 AI 요약 기능을 사용한 중복/복제(Scraped) 콘텐츠 사이트**로 판단하기 아주 쉬운 조건입니다.
* 또한 라이브스코어, 팀/선수 데이터 등 데이터베이스 중심의 콘텐츠는 구글 애드센스 정책상 '독창적인 정보 가치가 낮은 페이지'로 간주됩니다.

### 🚨 원인 ②: Cloudflare WAF 및 Bot Fight Mode 영향성 (직접 API 검증 결과: 이상 없음 ✅)
현재 사이트는 Cloudflare 프록시 서버를 통해 서빙되며, **Bot Fight Mode(`fight_mode: true`)**가 켜져 있어 자동화 봇에 대해 챌린지를 제기하는 환경입니다.
* **직접 API 조회 검증 결과**:
  프로젝트 내 Cloudflare API 키로 GraphQL 로그를 추출하여 **최근 23시간 동안의 실시간 방화벽 이벤트**를 긴급 분석했습니다:
  * **수집된 전체 구글/애드센스 크롤러 요청 수**: `836건` (`Mediapartners-Google`, `Googlebot` 등)
  * **WAF/방화벽에 차단 또는 챌린지(Challenge)된 구글 봇 건수**: `0건` (100% `Action: skip` 처리되어 통과함)
  * **구글 봇 대상 서버 응답 분배**: `200 OK` (822건), `301 Redirect` (11건), `304 Not Modified` (1건), `404 Not Found` (2건)
* **진단**: 현재 Cloudflare의 Custom WAF Ruleset 중 Verified Bot 패스 규칙(`cf.client.bot` 대상 skip 처리)이 구글 애드센스 심사 봇에도 정상 작동하고 있어, **Cloudflare 측 네트워크 단에서 구글 봇을 차단하는 문제는 없습니다.**

### 🚨 원인 ③: 프론트엔드 광고 스크립트의 부재 (Git 이력 역추적 결과 🔍)
* **Git 히스토리 분석 결과**:
  * **삭제 시점**: **2026년 6월 8일** (커밋 해시: `33d13e1b16d5da2e025306c97181561629309c64`, 작업명: "크롤링 테스트")
  * **삭제 사유 추정**: 이전 작업 이력인 "pagespeed 테스트"(`9801d229`) 등에서 모바일 로딩 지연 및 PageSpeed score 저하 이슈를 개선하거나, 광고 송출을 카카오 애드핏으로 전면 대체하는 과정에서 `src/app/layout.tsx` 헤더 내의 애드센스 로딩 스크립트가 완전히 제거된 것으로 확인되었습니다.
* **진단**: 
  현재 사이트에는 소유권 인증 메타 태그(`google-adsense-account`)만 존재하며, 실제 애드센스 심사 엔진이 레이아웃 분석 및 자동 광고 렌더링 검증을 진행할 때 필수적으로 요구하는 **`adsbygoogle.js` 공통 스크립트 로드 태그가 완전히 부재**합니다.
  구글 애드센스는 소유권 메타 태그로 1차 인증을 하더라도, 최종 사이트 준비 상태 심사 시에는 실제 스크립트 코드가 활성화되어 있어야 레이아웃 적합성 평가를 완료하고 승인을 해 줍니다. 이 스크립트가 로드되지 않기 때문에 최근 재도전 과정에서 계속 거절 통보가 내려왔을 가능성이 매우 높습니다.

## 2. 권장 해결 방안 (Action Items)

수정과 배포를 진행하지 않고 원인 파악만 요청하셨으므로, 아래 조치 사항을 준비하여 적용하시는 것을 권장합니다.

### 🛠️ 액션 1: Cloudflare WAF 규칙 검토 및 모니터링 (검증 완료 ✅)
* **상태**: API 로그 대조 결과 현재 정상 통과 중이므로 추가 우회 규칙 등록이 긴급하지는 않습니다.
* **권장 조치**: 추후 구글이 비정식 Compute Node(GCP) 대역에서 rDNS를 거치지 않고 애드센스 심사를 실행할 경우 `cf.client.bot` 분류를 타지 못해 차단당할 위험이 극소량 존재합니다. 이를 방비하기 위해, Cloudflare 대시보드 **[보안] > [WAF] > [사용자 지정 규칙]**에서 다음의 명시적 예외 규칙을 걸어두면 100% 안전합니다:
   * **필드**: `User Agent`
   * **연산자**: `contains`
   * **값**: `Mediapartners-Google`
   * **조치**: `우회 (Bypass)` -> 모든 Cloudflare 보안 기능 (특히 Bot Fight Mode) 우회

### 🛠️ 액션 2: 애드센스 기본 자바스크립트 스크립트 복구 및 성능 최적화 적용
애드센스 승인을 완료하기 위해서는 스크립트 로딩이 반드시 필요합니다. 단, PageSpeed 점수 하락을 방어하기 위해 Next.js 15의 최적화 전략을 결합해야 합니다.

1. **Next.js 15 최적화 스크립트 삽입 (`src/app/layout.tsx`)**:
   기존에 하드코딩되거나 누락되었던 스크립트를 `.env.local`의 환경변수(`NEXT_PUBLIC_ADSENSE_CLIENT_ID`)와 연동하여 헤더에 복구합니다.
   ```tsx
   import Script from 'next/script';

   // layout.tsx의 RootLayout 컴포넌트 내 <head> 영역에 삽입
   <Script
     async
     src={`https://pagead2.googlesyndication.com/pagead/js/adsbygoogle.js?client=${process.env.NEXT_PUBLIC_ADSENSE_CLIENT_ID}`}
     crossorigin="anonymous"
     strategy="afterInteractive" // 심사 단계에서는 신속한 감지를 위해 afterInteractive 권장
   />
   ```

2. **승인 이후 PageSpeed 스코어 방어 팁 (LCP 최적화)**:
   심사 완료 및 승인 후에는 LCP 점수를 극대화하기 위해 `strategy` 속성을 `lazyOnload`로 완화하거나, 브라우저가 유휴 상태일 때 로드되도록 변경하는 것을 권장합니다:
   * **대안**: `strategy="lazyOnload"` 설정 시 사용자 상호작용 및 Hydration이 끝난 뒤 백그라운드에서 지연 로드되므로 TBT(Total Blocking Time) 점수를 거의 깎지 않습니다.

### 🛠️ 액션 3: '가치 없는 콘텐츠' 우회를 위한 뉴스/커뮤니티 독창성 강화
게시글 수 대비 댓글이 현저히 부족한 구조를 개선해야 합니다.
1. 자동 RSS 수집 글 비율을 낮추고, **사람이 직접 쓴 축구 프리뷰, 칼럼, 경기 분석 글**의 노출 비율을 높입니다.
2. 구글봇이 크롤링할 때 사이트가 비어있어 보이지 않도록, 메인 페이지의 `BoardCollectionWidget`이나 `AllPostsWidget` 등에서 유효한 텍스트 콘텐츠가 확실히 pre-rendering(SSR)되어 전달되고 있는지 지속적으로 모니터링해야 합니다. (현재 SSR 자체는 정상 동작 중입니다.)
